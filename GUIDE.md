# 🎯 AI Code Review Platform: Comprehensive Operation Guide

Welcome to the **AI Code Review Platform**, a production-grade, FAANG-standard full-stack application designed for automated code quality analysis. This system leverages a high-availability infrastructure stack to provide real-time metrics, AI-driven refactoring suggestions, and deep observability.

---

## 🏗 System Architecture & Tech Stack

This project implements a classic **Six-Service Micro-Infrastructure** architecture:

1.  **Backend Core**: Spring Boot 3.x (Java 21) - Primary REST API and Business Logic.
2.  **Database**: PostgreSQL 16 - Relational storage for users, projects, and metadata.
3.  **Cache/Session**: Redis 7 - Highly performant transient data storage.
4.  **Message Queue**: RabbitMQ 3 - Asynchronous task processing (AI Analysis triggers).
5.  **Monitoring**: Prometheus - Time-series metrics collection from Actuator.
6.  **Observability**: Grafana - Unified performance and health dashboards.

## 🚀 Golden Path: How to Run the Product

### 1. Prerequisites
Ensure the following are installed and running on your host machine:
- **JDK 17 or 21**
- **Maven 3.9+**
- **Docker Desktop** (with Docker Compose)
- **Node.js 20+** (for Frontend testing)

### 2. Build and Launch
Follow these steps in your terminal to start the full ecosystem:

```powershell
# Step A: Package the Spring Boot binary
./mvnw clean package -DskipTests

# Step B: Orchestrate the Infrastructure
# This command builds the app image and starts all 6 services
docker-compose up -d --build
```

### 3. Verification
Run the following to ensure the backend is healthy:
```bash
curl http://localhost:8080/actuator/health
# Expected Response: {"status":"UP"}
```

## 📊 Monitoring & Observability

The platform is pre-instrumented with **Prometheus** and **Grafana**. 

- **Prometheus Metrics**: Available at `http://localhost:9090` (Target: `back-end`).
- **Grafana Dashboard**: Access at `http://localhost:3000` (Default: `admin/admin`).
    - *Dashboard Hint*: Import the "Spring Boot Observability" dashboard to see real-time JVM, CPU, and DB Connection Pool metrics.

## 🔐 Security Standards
- **Stateless Tokens**: RSA-signed JWTs (RS256) are used for all authenticated internal communication.
- **Service Isolation**: All infrastructure components (DB, MQ, Cache) are isolated within the `ai-code-review_default` network and are exposed only via authorized ports.
